Core features

Built around the actual flow of the project.

Face recognition

Webcam frames are analyzed and compared against stored student face embeddings so recognized students can be identified during check-in.

Attendance logging

Confirmed matches are written to a local attendance file with timestamps, keeping the record simple, readable, and easy to audit.

Student management

A compact desktop GUI lets staff add or remove students and keep the student roster aligned with face image filenames.

Local-first setup

Student records, logs, and attendance files live in the project itself, which makes the app easy to inspect, test, and extend.

Workflow

Three steps from roster to recorded attendance.

01

Build the student roster

Add names, classes, and face image references through the GUI so the project has a roster to compare against.

02

Run recognition

The webcam feed is scanned for faces, embeddings are generated, and each student match is checked against the attendance log for the day.

03

Store the result

Present students are timestamped in the attendance file, while logs provide a clear trail for troubleshooting and future improvements.
